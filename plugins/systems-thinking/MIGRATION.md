# systems-thinking Migration Notes

This file records the path and runtime-assumption adaptations made while
migrating the standalone upstream repository
`https://github.com/fakoli/systems-thinking-plugin` into the marketplace plugin
layout at `plugins/systems-thinking`.

The goal is to preserve upstream behavior while making copied definitions,
helpers, tests, assets, and docs resolve from the marketplace checkout.

## Source And Target Layout

| Area | Upstream location or assumption | Marketplace location or assumption | Notes |
| --- | --- | --- | --- |
| Repository root | `systems-thinking-plugin/` | `plugins/systems-thinking/` | All plugin-local paths resolve from the marketplace plugin directory. |
| Plugin identity | `systems-thinking-plugin` | `systems-thinking` | Marketplace installable plugin name. Upstream provenance is retained in README. |
| Plugin manifest | `.claude-plugin/plugin.json` with repository `fakoli/systems-thinking-plugin` | `plugins/systems-thinking/.claude-plugin/plugin.json` with repository `fakoli/fakoli-plugins` | Manifest stays plugin-local; aggregate marketplace metadata is generated from it. |
| Marketplace index | Upstream `.claude-plugin/marketplace.json` in standalone repo | Root `.claude-plugin/marketplace.json`, `registry/index.json`, and `registry/categories.json` in `fakoli-plugins` | Generated by `./scripts/generate-index.sh`; upstream standalone marketplace file is not copied. |
| Release version marker | Upstream `VERSION`, `pyproject.toml`, and `.claude-plugin/plugin.json` in standalone repo | `plugins/systems-thinking/VERSION`, `plugins/systems-thinking/pyproject.toml`, `plugins/systems-thinking/.claude-plugin/plugin.json`, and generated marketplace registry metadata | These are kept in sync for each marketplace package change. |
| Standalone setup | `setup.sh`, `.github/workflows/*`, `.gitignore`, `.prettierrc`, `.python-version` | Marketplace shared scripts and GitHub workflows at repo root | Standalone repository scaffolding is intentionally not imported into the plugin directory. |

## Hook Adaptations

All copied hook files moved from upstream `hooks/` to
`plugins/systems-thinking/hooks/`.

| File | Upstream path or assumption | Marketplace path or assumption |
| --- | --- | --- |
| `hooks/hooks.json` | Hook scripts are plugin-local shell commands | Commands use `${CLAUDE_PLUGIN_ROOT}/hooks/*.sh` so the installed plugin root resolves correctly. |
| `hooks/discover-components.sh` | Plugin invocation prefix `systems-thinking-plugin:` | Plugin invocation prefix `systems-thinking:` |
| `hooks/discover-components.sh` | Discover components under the current plugin root | Uses `${CLAUDE_PLUGIN_ROOT}` when set, otherwise resolves parent of the script directory. |
| `hooks/discover-components.sh` | Skill and agent lists from `skills/` and `agents/` | Skill and agent lists from `plugins/systems-thinking/skills/` and `plugins/systems-thinking/agents/` via `PLUGIN_ROOT`. |
| `hooks/user-prompt-gate.sh` | Current prompt payload field `.user_prompt` | Supports current `.prompt` with fallback to `.user_prompt`. |
| `hooks/stop-quality-gate.sh` | Quality checks scan the whole transcript | Prefer `.last_assistant_message` when available, fallback to transcript scanning. |

## Validation Commands

Use these commands from the marketplace repository root after changing
migration-sensitive paths:

```bash
test -f plugins/systems-thinking/MIGRATION.md && grep -qi 'adapt' plugins/systems-thinking/MIGRATION.md
./scripts/validate.sh plugins/systems-thinking
./scripts/test-path-resolution.sh plugins/systems-thinking
./scripts/check-changelogs.sh
./scripts/check-registry-drift.sh
(cd plugins/systems-thinking && uv run pytest tests --collect-only -q)
```
