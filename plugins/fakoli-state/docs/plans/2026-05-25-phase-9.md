## Context

PR #49 (Phase 8 ‚Äî bidirectional sync, v1.8.0) merged with 2 deferred CONSIDER items from the critic and several SHOULD FIX items logged but not fixed. The deferred items concern **audit-event honesty** ‚Äî the sync engine currently emits `sync.pull.completed` for branches where the local state was not actually updated (the resolution was deferred to the next pass). This makes the JSONL audit log lie about what happened. Phase 9 closes that loop.

Phase 7 also left three small cleanup items: the `RecordedLLMProvider` test double ignores `max_tokens`/`temperature` when computing its record key (so two recorded responses that differ only in those args silently collide), the brainstorm skill detection is fuzzy, and the `expand --use-llm` command has no `--format prd` UX for pasting back into `prd.md`.

The user additionally requested two new plugin-owned documentation agents:
- **`marketplace-scribe`** ‚Äî keeps marketplace-level artifacts in sync (`.claude-plugin/marketplace.json`, `registry/*.json`, root `README.md` plugin table) whenever the plugin's manifest or surface changes.
- **`docs-scribe`** ‚Äî keeps the plugin's own `docs/` folder fresh: audits cross-references between docs, ensures `CHANGELOG.md` and `plugin.json.description` match the code's actual shape, and flags stale references.

These agents are **created** in this phase; they become invocable as subagent types in future sessions (Claude Code discovers agents at session start).

### T3 ‚Äî Design `SyncAuditPayload` discriminated union

**Intent:** Replace the single all-optional `SyncAuditPayload` with a discriminated union (one concrete subclass per `sync.*` action), so unknown fields and field/action mismatches fail Pydantic validation instead of being silently accepted.

**Acceptance:**
- New per-action models in `bin/src/fakoli_state/state/payloads.py`:
  - `SyncPushStartedPayload`, `SyncPushCompletedPayload`, `SyncPushDeferredPayload`, `SyncPushFailedPayload`
  - `SyncPullStartedPayload`, `SyncPullCompletedPayload`, `SyncPullDeferredPayload`, `SyncPullFailedPayload`
  - `SyncReconciliationStartedPayload`, `SyncReconciliationCompletedPayload`
- Each subclass declares only the fields its action actually carries (`extra="forbid"`).
- Backwards-compat alias `SyncAuditPayload = Annotated[Union[...], Field(discriminator="action")]` kept so existing callers do not have to change at import time.
- Mapping table `ACTION_TO_PAYLOAD` exported.

**Scope:**
- `plugins/fakoli-state/bin/src/fakoli_state/state/payloads.py` (edit only the SyncAudit section)
